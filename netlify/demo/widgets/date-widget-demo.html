Date Widget Demo

Overview

The Date Widget renders a date picker for selecting dates. Users can pick a date from a calendar interface.

Key Features:

  • Native date picker
  • Min/max date constraints
  • Custom label text
  • Enter key submission
  • Browser-native calendar UI

Backend Response Format

To trigger a date widget, the backend should respond with:

{ "text": "When would you like to schedule your appointment?", "widget": { "type": "date", "label": "Select Date", "minDate": "2024-01-01", "maxDate": "2024-12-31", "buttonText": "Schedule" } }

Widget Data Structure

{ "type": "date", "label": "Select a date", "minDate": "YYYY-MM-DD", "maxDate": "YYYY-MM-DD", "buttonText": "Submit" }

Fields:

  • type: Must be "date"
  • label: Label text for the date picker (default: "Select a date")
  • minDate: Minimum selectable date in YYYY-MM-DD format (optional)
  • maxDate: Maximum selectable date in YYYY-MM-DD format (optional)
  • buttonText: Text displayed on the submit button (default: "Submit")

Usage Examples

Appointment Scheduling

{ "type": "date", "label": "Select Appointment Date", "minDate": "2024-01-01", "maxDate": "2024-12-31", "buttonText": "Schedule" }

Birthdate Selection

{ "type": "date", "label": "Select Your Birthdate", "maxDate": "2006-01-01", "buttonText": "Submit" }

Deadline Selection

{ "type": "date", "label": "Select Deadline", "minDate": "2024-01-01", "buttonText": "Set Deadline" }
